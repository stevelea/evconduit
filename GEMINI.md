## Project Purpose

`evconduit-backend` acts as a proxy between the HomeAssistant component `evconduit-homeassistant` and the Enode API. Its primary goal is to enable shared access to the Enode API for multiple users, eliminating the need for individual users to obtain their own Enode API keys. The project supports all vehicles compatible with Enode, with a specific focus on XPENG and NIO.

Users connect to EVConduit, generate an API key, and link their vehicle between EVConduit and Enode. Vehicle data can then be consumed by HomeAssistant either through API polling or via webhooks for real-time updates.

Future plans include subscription tiers (Free, Basic, Pro with a 30-day Pro trial) and value-added features like an "Insights" page for community vehicle statistics.

## Technical Stack

### Backend
* **Framework:** FastAPI
* **Language:** Python 3.12
* **Directory Structure:**
    * `api/`: Contains API endpoints.
    * `storage/`: Handles Supabase-related logic.
    * `services/`: Integrations with third-party services (e.g., Brevo, Twilio, Stripe). Enode integration is planned to be moved here.

### Frontend
* **Framework:** Next.js 15.3.1
* **Language:** TypeScript
* **Routing:** App Router
* **Structure:** `src` directory
* **UI Components:** ShadCN
* **Styling:** Tailwind CSS 4
* **Code Quality:** Linting enforced.

### Key Services & Integrations
* Resend
* Brevo
* Stripe
* Sentry
* Umami
* Supabase (for database, authentication, etc.)
